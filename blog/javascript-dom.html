Loops

Loops in JavaScript, such as for or while loops, enable repetitive execution of a block of code.
We will use a for loop as an example...

The variable slicesOfBread = 3 - meaning we have 3 slices of bread.
There are 3 parts inside the brackets of the for loop which are separated by a semicolon ; The first part in the bracket is called the initialization (let i = 0;) where we tell the computer we want to start our for loop at 0 and that 'i' is basically our loop control variable.

The second part of of the bracket is called the condition. This condition takes in a number or the length or an array. For this example we have 3 slices of bread so we put i < slicesOfBread, meaning i < 3 so that the loop will only execute 3 times. if we change slicesOfBread = 5 this means the loop will now execute 5 times which is why variables are important rather than hard coding the number in. If we didn't specify how many times we want the for loop to run it would infinitely run and most likely crash the computer :(

The third part of the bracket is called the Iteration Expression. This controls how the variable changes after each loop. i++ means to increment by one after a loop has completed, for example... the text below the code is the outcome of the code being run. We console log "Spread peanut butter on slice " + (i + 1); You can see that the code has been run 3 times which is what we specified in the condition part of the for loop bracket. But how has the number 1, 2 and 3 been put beside the text, first iteration of this loop i = 0 but then after the loop has completed and starts its second loop i = 1 since we incremented it (i++) by one every loop. Usually we do i + 1 if we are counting because the index starts from 0 so the first loop would be "Spread peanut butter on slice 0" which we don't want. We would only increment by just i if we were to do an array since the index of an array starts from 0.
